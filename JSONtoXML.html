The API Platform enables developers to convert messages from the JavaScript object notation (JSON) format to the extensible markup language (XML) format by using the JSONtoXML policy type. This policy is useful for enabling backend XML services to support RESTful apps that require JSON (for example, due to a lack of an XML parsing capabilities on the client).

In a typical mediation scenario, a JSONtoXML policy on the inbound request flow is often paired with an XMLtoJSON policy on the outbound response flow. By combining policies this way, a JSON API can be exposed for services that natively support only XML. For scenarios where APIs are consumed by diverse clients apps which may require either JSON and XML, the format of the response can be dynamically set by configuring JSONtoXML and XMLtoJSON policies to execute conditionally. See the API Platform Developer Guide topic [node:4583] for an implementation of this scenario.

The HTTP Content-type header of the source message must be set to application/json. If the HTTP Content-type header of the source message is not application/json, the policy is not enforced. The payload of the JSON message is parsed and converted into XML, and the HTTP Content-type header of the XML-formatted message is set to application/xml.

Configure the JSONtoXML policy using the following optional elements. Reasonable defaults are provided for each setting, thus all elements in this policy type are optional unless NamespaceBlockName is set, in which case DefaultNamespaceNodeName and NamespaceSeparator must also be specified.

Field Name Description
Source The variable containing the JSON-formatted message to be converted to XML.
  • If source is missing, it is treated as a simple message. For example, <Source>message</Source>
  • If the source variable cannot be resolved, or resolves to a non-message type, the policy execution fails.
Output variable
The variable name of the resultant XML-formatted message.
If OutputVariable is not specified, then the source variable is treated as OutputVariable.
Invalid character replacement
Replaces any JSON elements that are invalid in XML with the defined string. For example:
{"this is%%%invalid" : "value1/value2"}
converts to:
<this_is_invalid>value1value2</this_is_invalid>
Text node name
Converts the property into XML text. For example:
{
 "root" : {
 "one": "value1",
 "#text": "value0",
 "two" : "value2"}
}
converts to:
<root>
  <one>value1</one>value0<two>value2</two>
</root>
If TextNodeName is not specified, this XML is generated:
<root><one>value1</one><_text>value0</_text><two>value2</two></root>
Null value
Indicates a null value. By default the value is NULL. For example, where I_AM_NULL is configured for the Null value:
{"root" : "I_AM_NULL"}
converts to:
<root></root>
Where no value (or a value other than I_AM_NULL) is specified for the Null value, then the same payload converts to:
<root>I_AM_NULL</root>
Attribute block name
Converts the property into XML attributes. For example:
{
"root" : 
{
   "#attrs" : 
{
   "@att1" : "value1",
   "@att2" : "value2"
   "att3" : "value3"
 }, 
   "child" : "value0",
 }
}
converts to:
<root att1="value1" att2="value2"><child>value0</child></root>
Note: The property "att3" : "value3" is ignored as it does not start with the @ symbol.
Attribute prefix
Converts the property starting with the specified prefix into XML attributes. For example:
{
 "root" : {
   "@att1" : "value1",
   "child" : "value0",
   "@att2" : "value2"
 }
}
converts to:
<root att1="value1" att2="value2"><child>value0</child></root>
Namespace block name
Configures XML namespace properties. For example:
{
 "root" : {
   "#namespaces" : 
{
   "$default" : "http://default",
   "xmlns:ns1" : "http://ns1"
 },
   "child1" : "value0",
   "ns1:child2" : "value1"
 }
}
converts to:
<root xmlns="http://default" xmlns:ns1="http://ns1"><child1>value0</child1><ns1:child2>value1</ns1:child2></root>
Default namespace node name
Namespace separator
Array root element name
Configures an XML array and item. For example:
[ "one", { "two": 2 }, "three" ]
converts to:
<Array>
   <Item>one</Item>
   <Item> <two>2</two></Item>
   <Item>three</Item>
</Array>
